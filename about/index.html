Governance and Dignity.

We deploy layer 3&4 ToIP governance frameworks for a restricted domain (a tall, thin vertical). the restriction enables a nice, clean GF to be written and reduces the number of credential definitions (schema) involved to managable levels. It uses a common data store from which it serves (a) the GF, (b) schemas, and (c) directory services - and optionally can serve as (d) a credential wallet. Access to the environment comes with registering a public key with the GF, according to the GF rules (e.g. signing the GF).
